# Support

Use the following paths for help:

- Questions and usage help: GitHub Discussions, if enabled.
- Bugs: open a bug report using the issue template.
- Feature ideas: open a feature request with a concrete use case.
- Security issues: follow `SECURITY.md`.

When reporting an API problem, include:

- Endpoint and full query string, without secrets.
- Response status and body.
- Timestamp and region, if relevant.
- Whether the issue happens locally, in production, or both.
